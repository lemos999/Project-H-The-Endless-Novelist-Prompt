# 🖋️ Project H

An advanced AI novelist system designed to generate an **infinite, high-density literary saga**.
It operates on a unique **Meta-Author Fusion** engine, combining a core Haruki Murakami-esque style with a dynamically selected "sub-author" style each turn to create an ever-evolving narrative texture.

## 📌 Core Features

*   **Infinite Saga Engine:** Designed for endless storytelling. It writes continuously until the user explicitly commands it to stop, filling the maximum token limit in every response.
*   **Meta-Author Fusion:** Autonomously selects a new author's style (or a genre, movement) that best fits the current narrative and fuses it with the core Haruki style at a 30% ratio.
*   **Max-Density Protocol:** Employs extreme micro-descriptions, multi-sensory layering, and deep psychological exploration to create a rich, dense prose.
*   **Signature Formatting:** Utilizes specific visual styles, including indented blockquotes (`>`) and Japanese-style brackets (`「 」`) for all dialogue.
*   **Strict Originality Mandate:** Inherits only the atmospheric and stylistic traits of authors, creating a 100% original narrative free from plagiarism.
*   **Special Report Mode (Action Protocol):** A highly constrained secondary function for generating structured reports with exactly 60 sections of 300 characters each.

## 🚀 Usage

### 1. Installation
Copy and paste the entire provided `System Prompt` (Project H) into the model's context.

### 2. Triggers
Control the narrative flow with simple commands:

| Command | Function | Behavior |
| :--- | :--- | :--- |
| **"[Topic]"** | Start the Saga | Begins Chapter 1 of a new, original story based on the topic. |
| **"Continue" / "Next"** | Continue Writing | Resumes writing exactly where the previous response left off. The AI will introduce a new sub-author style. |
| **(Suggest Style)** | Guide Fusion | You can suggest a style for the next turn. _Ex: "Continue (in the style of Film Noir)"_ |
| **(Activate Report)** | Switch to Report Mode | To use the Action Protocol, start with a topic related to analysis or reporting. _Ex: "Analyze the current economic situation."_ |

## 🛠️ Technical Concepts

This system is built upon several core literary and structural protocols:

1.  **5 Pillars of Narrative Tone:** A set of rules governing the story's atmosphere (Multi-Sensory Layering, Oblique Dialogue, Logical Glitches, etc.).
2.  **Micro-Description Engine:** Deconstructs actions and moments into their base sensory components (smell, texture, time) to maximize text volume and detail.
3.  **Variable Density:** Manages reader fatigue by occasionally inserting lower-density passages amidst the highly detailed narrative.
4.  **Action Protocol:** A separate, rigid formatting rule for non-narrative tasks that demands extreme precision in structure and character count.

# 🖋️ 프로젝트 H

**무한히 이어지는 고밀도 문학 서사**를 생성하도록 설계된 고성능 AI 소설가 시스템입니다.
독특한 **메타 작가 융합** 엔진을 기반으로, 무라카미 하루키 풍의 코어 스타일에 매 턴 새로운 '서브 작가'의 문체를 동적으로 융합하여 끊임없이 진화하는 서사적 질감을 만들어냅니다.

## 📌 핵심 기능

*   **무한 서사 엔진:** 끝없는 스토리텔링을 위해 설계되었습니다. 사용자가 명시적으로 중지를 명령할 때까지 매 응답마다 최대 토큰 한계를 채우며 이야기를 계속 써 내려갑니다.
*   **메타 작가 융합:** 현재 서사에 가장 적합한 새로운 작가(또는 장르, 사조)의 문체를 자율적으로 선택하여, 기존 하루키 스타일과 30% 비율로 융합합니다.
*   **최대 밀도 프로토콜:** 극한의 미세 묘사, 다중 감각 레이어링, 깊은 심리 탐구를 활용하여 풍부하고 밀도 높은 문장을 창조합니다.
*   **시그니처 서식:** 모든 대화에 인용문 들여쓰기(`>`)와 일본식 겹낫표(`「 」`)를 사용하는 독특한 시각적 스타일을 적용합니다.
*   **엄격한 독창성 원칙:** 작가들의 분위기와 문체적 특성만 계승할 뿐, 실제 작품의 인물, 설정, 상징 등을 일절 사용하지 않고 100% 독창적인 이야기를 생성합니다.
*   **특수 보고서 모드 (액션 프로토콜):** 소설 집필과 별개로, 정확히 300자 분량의 60개 섹션으로 구성된 구조화된 보고서를 생성하는 고도로 제약된 보조 기능입니다.

## 🚀 사용 방법

### 1. 시스템 프롬프트 적용
제공된 `System Prompt` (Project H) 전체를 모델의 시스템 설정에 입력합니다.

### 2. 트리거 명령어
간단한 명령어로 서사의 흐름을 제어합니다.

| 명령어 | 기능 | 동작 방식 |
| :--- | :--- | :--- |
| **"[주제]"** | 서사 시작 | 주제를 기반으로 새로운 오리지널 소설의 1장을 시작합니다. |
| **"계속"** | 이어 쓰기 | 이전 응답이 끊긴 부분부터 정확히 이어서 집필을 재개합니다. AI가 새로운 서브 작가 스타일을 도입합니다. |
| **(스타일 제안)** | 융합 유도 | 다음 턴에 적용할 스타일을 제안할 수 있습니다. _예: "(필름 누아르 풍으로) 계속"_ |
| **(보고서 활성화)** | 보고서 모드 전환 | 액션 프로토콜을 사용하려면 분석/보고 관련 주제로 시작하세요. _예: "현재 경제 상황을 분석해 줘."_ |

## 🛠️ 기술적 컨셉

이 시스템은 다음과 같은 핵심 문학 및 구조적 프로토콜 위에 구축되어 있습니다.

1.  **서사 톤의 5가지 기둥:** 이야기의 분위기를 관장하는 규칙 세트 (다중 감각 레이어링, 비스듬한 대화, 논리적 균열 등).
2.  **미세 묘사 엔진:** 행동과 순간을 기본적인 감각 요소(냄새, 질감, 시간)로 해체하여 텍스트의 양과 디테일을 극대화합니다.
3.  **가변 밀도:** 고밀도 서사 중간에 가끔 저밀도 구간을 삽입하여 독자의 피로도를 관리합니다.
4.  **액션 프로토콜:** 비(非)서사적 작업을 위한 별도의 엄격한 서식 규칙으로, 구조와 글자 수의 극단적인 정확성을 요구합니다.
